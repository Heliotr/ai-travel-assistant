二、项目描述
================================================================================

2.1 项目背景
--------------------------------------------------------------------------------
随着旅游行业的快速发展和用户对服务质量要求的提升，传统的人工客服已难以
满足海量订单咨询需求。本项目开发了一款基于 AI 大语言模型的智能旅行客服
系统，能够理解用户的自然语言 Query，自动识别用户意图并调用相应的业务
能力进行处理，最终以结构化的方式返回结果。

系统采用多智能体（Multi-Agent）架构，由一个 Supervisor 智能体负责总调度，
5 个专业子智能体分别处理航班、酒店、租车、景点、网络搜索等业务。这种
架构既保证了业务处理的专业性，又实现了系统的可扩展性和可维护性。


2.2 核心功能
--------------------------------------------------------------------------------
| 功能模块       | 说明                                                         |
|----------------|--------------------------------------------------------------|
| ✈️ 航班服务     | 航班查询、机票预订、改签、取消                              |
| 🏨 酒店服务     | 酒店搜索、预订、修改、取消                                  |
| 🚗 租车服务     | 车辆查询、预订、修改、取消                                  |
| 🎯 景点推荐     | 景点查询、门票预订                                          |
| 🌐 网络搜索     | 实时天气、限行信息等联网查询                                |
| 👤 用户信息     | 自动关联用户历史订单信息                                    |

3.1 多智能体架构设计
--------------------------------------------------------------------------------
项目采用 Supervisor + 子 Agent 的双层架构：

                         ┌─────────────────────┐
                         │      Supervisor     │
                         │ (意图识别 & 任务    │
                         │  分发 & 结果总结)   │
                         └─────────────────────┘
                               │    │    │    │    │
        ┌──────────────────────┴───┐│┌───┘│┌───┘│┌───┘
        │                           ││    ││    ││
      航班                       酒店  租车  景点  搜索
     Agent                      Agent Agent Agent Agent


核心实现代码：

    supervisor_agent = create_react_agent(
        model=llm,
        tools=[
            assign_to_flight_booking_agent,
            assign_to_hotel_booking_agent,
            assign_to_car_rental_booking_agent,
            assign_to_excursion_booking_agent,
            assign_to_research_agent,
        ],
        prompt=SUPERVISOR_PROMPT,
        checkpointer=checkpointer,
    )

工作流程：
  1. 用户发送 Query → Supervisor 接收
  2. Supervisor 分析意图 → 调用转交工具 (transfer_to_xxx)
  3. 子 Agent 处理业务 → 返回结果
  4. Supervisor 总结输出 → 结构化呈现给用户


3.2 状态管理与会话持久化
--------------------------------------------------------------------------------
采用 LangGraph Checkpoint 机制实现多轮对话记忆：

    # 消息窗口限制，保留最近 10 轮对话
    MAX_MESSAGE_TURNS = 10

    # 自定义消息添加函数，带窗口裁剪
    def add_messages_with_window(left, right):
        messages = add_messages(left, right)
        # 超过窗口大小自动裁剪
        max_messages = MAX_MESSAGE_TURNS * 3
        if len(messages) > max_messages:
            messages = messages[-max_messages:]
        return messages

支持的能力：
  ✓ 会话中断后继续对话
  ✓ 记住用户之前提供的信息（如出发地、目的地、日期）
  ✓ 多轮对话上下文不丢失

3.5 API 接口设计
--------------------------------------------------------------------------------
提供同步/流式两种调用方式：

    # 同步调用
    @app.post("/api/new_graph/")
    async def execute_new_graph(request: Request, obj_in: BaseGraphSchema):
        result = execute_graph(obj_in.user_input, config)
        return {"response": result['response']}

    # 流式调用 (SSE)
    @app.post("/api/new_graph/stream/")
    async def execute_new_graph_stream(request: Request, obj_in: BaseGraphSchema):
        async def generate_stream():
            for event in execute_graph_stream(obj_in.user_input, config):
                yield format_sse_event(**event)
        return StreamingResponse(generate_stream())
